📊 Query 3A: Worker Utilization Matrix

Multi-dimensional analysis of worker productivity and equipment utilization

🔍 What This Query Shows:

This complex analytical query provides a comprehensive view of worker productivity by measuring:

  • Assigned Shifts: Number of shifts assigned to each worker
  • Total Assigned Hours: Sum of shift durations
  • Equipment Types Used: Diversity of equipment operated
  • Actual Hours Worked: Real hours logged in usage records
  • Utilization Percentage: Actual vs Assigned hours ratio
  • Incident Rate: Incidents per shift (safety indicator)
  • Performance Level: Classification based on utilization

SQL Complexity: Multiple JOINs, GROUP BY, Window Functions (RANK, PERCENT_RANK), Aggregations

🔍 SQL Query Breakdown

SELECT 
    w.WorkerID, w.Name, w.Role,
    COUNT(DISTINCT asn.ShiftID) AS AssignedShifts,
    SUM(s.Duration) AS TotalAssignedHours,
    COUNT(DISTINCT ur.EquipmentID) AS EquipmentTypes,
    SUM(ur.HoursUsed) AS ActualHoursWorked,
    ROUND(SUM(ur.HoursUsed) / SUM(s.Duration) * 100, 2) AS UtilizationPercent,
    RANK() OVER (ORDER BY SUM(ur.HoursUsed) DESC) AS ProductivityRank
FROM Worker w
LEFT JOIN Assigned_Shift asn ON w.WorkerID = asn.WorkerID
LEFT JOIN Shift s ON asn.ShiftID = s.ShiftID
LEFT JOIN Usage_Record ur ON w.WorkerID = ur.WorkerID
WHERE w.EmploymentStatus = 'Active'
GROUP BY w.WorkerID, w.Name, w.Role
ORDER BY ActualHoursWorked DESC;

Key Concepts:

  • Multiple LEFT JOINs: Connects workers to shifts and usage records
  • DISTINCT: Counts unique items (shifts, equipment)
  • Window Function RANK(): Ranks workers by productivity
  • Calculated Fields: Utilization percentage formula
  • WHERE Clause: Filters active employees only
